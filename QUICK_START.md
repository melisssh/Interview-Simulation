# Quick Start Guide - Ollama Integration Complete ✅

## What Changed?
All old APIs (Gemini, OpenAI, Groq) have been replaced with **Ollama** (local LLM).

### Changed Functions:
1. ✅ `generate_questions_with_ai()` - Question generation now uses Ollama
2. ✅ `/interviews/{interview_id}/chat` - Coaching endpoint now uses Ollama  
3. ✅ `/ws/interview/{interview_id}` - WebSocket was already using Ollama (verified clean)
4. ✅ Removed all orphaned Groq code

## Running the System

### Step 1: Start Ollama
```bash
ollama serve
# Keep this running in a terminal
```

### Step 2: Ensure Model is Available
```bash
ollama pull llama3.2:3b
```

### Step 3: Start Backend
```bash
cd backend
pip install -r requirements.txt
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### Step 4: Start Frontend
```bash
cd frontend
npm install
npm run dev
# Opens at http://localhost:5173
```

## What You Can Do Now

### ✅ Create Interview (REST API)
```bash
POST http://localhost:8000/interviews
{
  "position": "Backend Developer",
  "company_name": "Trendyol",
  "department_name": "Core Backend",
  "domain": "general",
  "language": "tr",
  "cv_text": "Python, FastAPI, SQL experience...",
  "profile_university": "Yıldız Teknik Üniversitesi",
  "profile_department": "Bilgisayar Mühendisliği",
  "profile_class_year": "4"
}
```
→ Ollama generates 6 interview questions

### ✅ Real-Time Interview (WebSocket)
```
ws://localhost:8000/ws/interview/{interview_id}

1. Send: {"type": "init", "domain": "general", "max_questions": 5}
2. Receive: Opening question from Ollama
3. Send: {"type": "audio", "audio": "base64_encoded_wav"}
4. Receive: Transcription + Next question from Ollama
5. Repeat until done
6. Send: {"type": "end"}
```

### ✅ Coaching Chat (REST API)
```bash
POST http://localhost:8000/interviews/{interview_id}/chat
{
  "message": "How can I improve my communication?"
}
```
→ Ollama provides coaching based on your interview

## Troubleshooting

### "Ollama connection refused"
```
Error: Failed to connect to Ollama at localhost:11434
Fix: ollama serve
```

### "Model llama3.2:3b not found"
```
Error: model "llama3.2:3b" not found
Fix: ollama pull llama3.2:3b
```

### "Questions are empty/invalid"
```
Symptom: No questions generated or JSON parse errors
Check: 
1. Ollama is running (curl http://localhost:11434/api/tags)
2. Model is loaded (ollama ps)
3. Backend logs for "OLLAMA QUESTION GENERATION ERROR"
```

### "WebSocket connection fails"
```
Symptom: Can't connect to ws://localhost:8000/ws/interview/{id}
Check:
1. Backend is running on port 8000
2. Interview was created successfully
3. No CORS errors in browser console
```

### "STT not working"
```
Symptom: Audio sent but no transcription
Check:
1. Microphone permission granted
2. Audio is being recorded (check browser volume)
3. Faster Whisper model loaded in backend logs
```

## Performance Notes

⚠️ **This is CPU-based inference** (local):
- Question generation: ~2-5 seconds
- STT transcription: ~1-2 seconds per 10s audio
- LLM response: ~2-4 seconds
- All inference happens locally (no network)

**Speed tips:**
- Faster CPU = faster responses
- GPU acceleration available (future optimization)
- Questions may be slower on first run (model warmup)
